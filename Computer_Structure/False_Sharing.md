# 거짓 공유
일관성 있는 캐시를 가진 시스템에서 캐싱 메커니즘에 의해 관리되는 가장 작은 리소스 블록 크기에서 발생할 수 있는 성능저하 사용패턴. <br/>

## 발생하는 이유
캐시에 저장되는 가장 작은 단위, 캐시라인은 보통 64바이트 (혹은 32바이트나 128바이트 등)의 크기를 갖는다. <br/>
같은 캐시라인에 있는 데이터가 수정된 경우, 해당 캐시 데이터가 원본과 다르다고 판단하여 다시 메모리로부터 불러오는 작업을 거치게 된다. <br/>
즉, 인접한 메모리에 있는 데이터가 수정되었음에도 해당 데이터가 수정되었다고 판단하여 캐시미스가 발생하는 현상. <br/>

## 방지할 수 있는 방법
일반적으로 패딩을 추가하여 방지할 수 있다. <br/>
하나의 캐시라인에 하나의 데이터만 들어가게한다던지, 자주 사용되는 공유데이터를 캐시라인에서 분리하여 캐시미스의 확률을 줄이는 것.
